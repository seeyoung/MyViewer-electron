## 1. 문서 소개

### 1.1 목적

이 문서는 MyViewer-electron 프로젝트의 UI/UX 개선이 필요한 사항을 종합적으로 분석하고, 우선순위와 실행 계획을 제시합니다.

### 1.2 분석 범위

- **대상 컴포넌트**: 모든 렌더러 프로세스 UI 컴포넌트
- **평가 기준**: 사용성, 시각 디자인, 접근성, 인터랙션, 정보 구조, 반응성
- **분석 방법**: 코드 리뷰, 디자인 패턴 분석, 접근성 감사

### 1.3 문서 구조

이 개선 계획은 4개의 문서로 구성됩니다:

1. **UI-UX-OVERVIEW.md** (본 문서) - 전체 개요 및 우선순위

## 2. 전체 평가 요약

### 2.1 현재 상태

MyViewer-electron은 기능적으로 잘 구현된 이미지 뷰어 애플리케이션입니다.

**강점:**
- ✅ 핵심 기능 완성도 높음 (아카이브 열기, 이미지 탐색, 줌/팬)
- ✅ Electron + React 기반의 견고한 아키텍처
- ✅ 키보드 단축키 지원
- ✅ 다양한 뷰 모드 및 레이아웃 옵션
- ✅ 슬라이드쇼 기능

**개선 필요 영역:**
- ❌ 시각적 일관성 부족 (디자인 시스템 미구축)
- ❌ 접근성 문제 (ARIA, 색상 대비, 키보드 네비게이션)
- ❌ 인터랙션 피드백 부족 (애니메이션, 로딩 상태)
- ❌ UI 밀집도 높음 (특히 NavigationBar)
- ❌ 반응형 디자인 부족

### 2.2 핵심 문제점

#### 문제 1: 디자인 시스템 부재

**현상:**
- 색상 값이 컴포넌트마다 다름 (#444, #3d3d3d, #2d2d2d 등 유사 회색 톤 남발)
- 폰트 크기 일관성 없음 (1.3125rem, 1.275rem, 1.2rem, 1.125rem 등)
- 간격(spacing) 시스템 없음 (0.25rem, 0.4rem, 0.5rem, 0.75rem 등)
- 버튼 스타일이 컴포넌트마다 다름

**영향:**
- 유지보수 어려움
- 시각적 일관성 저하
- 전문성 부족한 인상

#### 문제 2: 접근성 기준 미달

**현상:**
- 색상 대비 WCAG AA 기준 미달 (#999 on #2d2d2d = 4.36:1, 요구: 4.5:1)
- ARIA 레이블 누락 (특히 아이콘 버튼)
- 키보드 포커스 표시 불명확
- 스크린 리더 지원 부족

**영향:**
- 접근성 법규 위반 가능성 (ADA, Section 508)
- 장애인 사용자 차별
- 잠재 고객층 배제

#### 문제 3: UI 밀집도 높음

**현상:**
- NavigationBar에 10개 이상의 컨트롤 집중
- 헤더에 최근 파일 5개까지 표시
- 사이드바, 썸네일, 슬라이드쇼 패널이 동시에 표시 가능

**영향:**
- 인지 부하 증가
- 학습 곡선 상승
- 좁은 화면에서 사용 불편

#### 문제 4: 사용자 피드백 부족

**현상:**
- 로딩 상태가 단순 텍스트 ("Loading...")
- 버튼 클릭 피드백 제한적
- 애니메이션 거의 없음 (일부 transition만)
- 에러 메시지 불친절

**영향:**
- 앱이 느리게 느껴짐
- 사용자 불안감 증가
- 전문성 저하

### 2.3 평가 점수

| 평가 기준 | 점수 | 설명 |
|----------|------|------|
| **사용성** | 6/10 | 기능은 있으나 학습 곡선 높음 |
| **시각 디자인** | 5/10 | 기능 중심, 디자인 시스템 부재 |
| **접근성** | 4/10 | 기본 키보드 지원만, ARIA 부족 |
| **인터랙션** | 5/10 | 기본 기능만, 피드백 부족 |
| **정보 구조** | 6/10 | 기능 추가로 복잡해짐 |
| **반응성** | 5/10 | 미디어 쿼리 거의 없음 |
| **전체** | **5.2/10** | 기능 완성도 높으나 UX 개선 필요 |

## 3. 우선순위별 개선 계획

### 3.1 긴급 (Critical) - 1주 이내

**법적/윤리적 이슈 해결**

| 항목 | 파일 | 예상 시간 | 영향도 |
|------|------|-----------|--------|
| 색상 대비 개선 | `index.css`, 모든 컴포넌트 | 4시간 | 높음 |
| ARIA 레이블 추가 | `NavigationBar.tsx`, `FolderSidebar.tsx`, `BottomThumbnails.tsx` | 6시간 | 높음 |
| 키보드 포커스 개선 | `index.css` | 2시간 | 중간 |

**상세 내용:** [UI-UX-ACCESSIBILITY.md](./UI-UX-ACCESSIBILITY.md) 참조

### 3.2 높음 (High) - 2-3주

**기초 인프라 구축 및 주요 UX 개선**

| 항목 | 파일 | 예상 시간 | 영향도 |
|------|------|-----------|--------|
| 디자인 시스템 구축 | `styles/design-tokens.css` (신규) | 8시간 | 매우 높음 |
| NavigationBar 리팩토링 | `NavigationBar.tsx` | 12시간 | 높음 |
| 로딩/에러 상태 개선 | `LoadingIndicator.tsx`, `ImageViewer.tsx` | 8시간 | 높음 |
| 툴팁 시스템 구현 | `components/shared/Tooltip.tsx` (신규) | 6시간 | 중간 |

**상세 내용:** [UI-UX-VISUAL-DESIGN.md](./UI-UX-VISUAL-DESIGN.md), [UI-UX-COMPONENTS.md](./UI-UX-COMPONENTS.md) 참조

### 3.3 중간 (Medium) - 4-6주

**사용자 경험 향상**

| 항목 | 예상 시간 | 영향도 |
|------|-----------|--------|
| 폴더 트리 개선 (확장/축소, 아이콘) | 10시간 | 중간 |
| 썸네일 가상 스크롤 | 12시간 | 중간 |
| 애니메이션 시스템 | 8시간 | 중간 |
| Welcome 화면 개선 | 6시간 | 낮음 |
| 반응형 레이아웃 | 10시간 | 중간 |

**상세 내용:** [UI-UX-COMPONENTS.md](./UI-UX-COMPONENTS.md) 참조

### 3.4 낮음 (Low) - 7주 이후

**고급 기능 및 개인화**

| 항목 | 예상 시간 | 영향도 |
|------|-----------|--------|
| 테마 시스템 (라이트/다크) | 16시간 | 중간 |
| 커스터마이징 옵션 | 12시간 | 낮음 |
| 고급 제스처 (핀치 줌, 스와이프) | 10시간 | 낮음 |
| 성능 최적화 (이미지 프리로딩 등) | 20시간 | 중간 |

## 4. 개선 로드맵

### Phase 1: 기초 다지기 (1-2주)

**목표:** 접근성 기준 충족 및 디자인 기초 구축

```
Week 1:
├─ 색상 대비 개선 (4h)
├─ ARIA 레이블 추가 (6h)
├─ 키보드 포커스 개선 (2h)
└─ 디자인 시스템 구축 시작 (8h)

Week 2:
├─ 디자인 시스템 완료 및 적용
└─ 접근성 테스트 및 검증
```

**산출물:**
- ✅ WCAG AA 기준 충족
- ✅ 디자인 토큰 시스템 (CSS 변수)
- ✅ 접근성 감사 보고서

### Phase 2: 핵심 경험 개선 (3-4주)

**목표:** 주요 컴포넌트 UX 개선

```
Week 3-4:
├─ NavigationBar 리팩토링 (12h)
├─ 로딩/에러 상태 개선 (8h)
├─ 툴팁 시스템 구현 (6h)
└─ 디자인 시스템 전체 적용 (10h)
```

**산출물:**
- ✅ 개선된 네비게이션 UI
- ✅ 친절한 로딩/에러 메시지
- ✅ 일관된 툴팁 시스템
- ✅ 통일된 시각 디자인

### Phase 3: 고급 기능 (5-6주)

**목표:** 사용자 경험 세밀화

```
Week 5-6:
├─ 폴더 트리 개선 (10h)
├─ 썸네일 가상 스크롤 (12h)
├─ 애니메이션 추가 (8h)
├─ Welcome 화면 개선 (6h)
└─ 반응형 레이아웃 (10h)
```

**산출물:**
- ✅ 더 나은 폴더 탐색
- ✅ 성능 최적화
- ✅ 부드러운 애니메이션
- ✅ 다양한 화면 크기 지원

### Phase 4: 정제 및 검증 (7-8주)

**목표:** 사용자 피드백 반영 및 최종 점검

```
Week 7-8:
├─ 사용자 테스트 (베타)
├─ 피드백 반영
├─ 접근성 최종 감사
├─ 성능 측정 및 최적화
└─ 문서화
```

**산출물:**
- ✅ 사용자 테스트 보고서
- ✅ 접근성 인증서 (VPAT)
- ✅ 성능 벤치마크
- ✅ UI/UX 가이드라인
