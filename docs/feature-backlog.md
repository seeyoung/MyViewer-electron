# Feature Backlog

현재 확인된 개선/추가 요청 항목들입니다.

## 확인된 아이디어

1. **Folder/Thumbnail Sidebar Enhancements**
   - [x] 폴더/썸네일 탭 전환
   - [x] 썸네일을 현재 폴더 기반으로 자동 동기화
   - [ ] 썸네일 무한 스크롤 (200장 이상 지원)
   - [ ] 썸네일에서 빠른 라벨/삭제 액션 지원

2. **휴지통 정리 기능**
   - [ ] 검색/필터 UI (예: 파일명 패턴, 날짜, 크기 등)
   - [ ] 정렬 옵션 (최신/오래된, 파일 크기 등)
   - [ ] "빠른 라벨링" 모드 (Restore / Delete permanently)

3. **성능/UX 개선**
   - [ ] 대용량 폴더 스캔 진행률 표시 + 취소 버튼
   - [ ] 백그라운드 스캔 (worker) 도입 검토

4. **책 넘기기(Viewer Double-Page) 모드**
   - [ ] `ViewMode.DOUBLE` 상태 추가, 현재 페이지를 짝수 인덱스로 고정하는 네비게이션 로직 필요
   - [ ] `ImageViewer`가 두 장(현재 페이지 + 다음 페이지)을 동시에 렌더링하도록 레이아웃/fit 계산 확장
   - [ ] Prev/Next, 키보드, 썸네일 클릭 시 두 페이지 단위로 이동하도록 수정 및 단일/이열 토글 UI 제공

5. **폴더/아카이브 자동 이동**
   - [ ] `useImageNavigation`이 마지막/첫 번째 이미지 도달 시 다음/이전 폴더(또는 최근 소스 목록)로 자동 전환하는 옵션 추가
   - [ ] 폴더 트리/최근 소스 간 순서를 명확히 정의하고, `goToNext`/`goToPrevious`에 graceful fallback 처리
   - [ ] 사용자 설정에서 “폴더 끝에서 자동 이동” 토글 제공 (예상치 못한 이동 방지)

6. **폴더 초기 로딩 지연 개선**
   - [ ] `folder:open` IPC를 “최소 이미지 + 초기 폴더 정보”만 먼저 반환하고, 나머지 하위 폴더 탐색은 백그라운드 작업으로 전환
   - [ ] 백그라운드 스캔 진행률/완료 이벤트(`folder:scan-progress` 등)를 추가하고, `FolderSidebar`가 이를 구독해 점진적으로 트리를 채움
   - [ ] 진행 중 취소/중단 로직과 UI(로딩 표시, “백그라운드 스캔 중” 메시지)를 마련해 UX 개선

추가 의견이나 우선순위 변경 사항이 있으면 이 문서를 기반으로 업데이트해 나가면 됩니다.
